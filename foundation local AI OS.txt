⭐ PART 1 — The REAL database schema

You want the rubric in the database, not hardcoded.
events table

Stores every event the OS detects.
sql

CREATE TABLE IF NOT EXISTS events (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  ts INTEGER NOT NULL,
  event_type TEXT NOT NULL,
  units REAL NOT NULL
);

event_types table

Stores the rubric: event → $/unit.
sql

CREATE TABLE IF NOT EXISTS event_types (
  event_type TEXT PRIMARY KEY,
  dollars_per_unit REAL NOT NULL

#[tauri::command]
fn log_event(state: tauri::State<AppState>, event_type: String, units: f64) {
    let conn = state.db.lock().unwrap();
    let ts = chrono::Utc::now().timestamp();

    conn.execute(
        "INSERT INTO events (ts, event_type, units) VALUES (?1, ?2, ?3)",
        params![ts, event_type, units],
    ).unwrap();
}

⭐ PART 3 — 15‑minute + YTD totals (the live ticker math)
rust

#[derive(serde::Serialize)]
struct Totals {
    last_15m: f64,
    ytd: f64,
}

#[tauri::command]
fn get_totals(state: tauri::State<AppState>) -> Totals {
    use chrono::{Utc, Datelike};

    let conn = state.db.lock().unwrap();
    let now = Utc::now().timestamp();
    let fifteen_ago = now - 15 * 60;

    // 15m
    let last_15m: f64 = conn.query_row(
        "
        SELECT COALESCE(SUM(e.units * t.dollars_per_unit), 0)
        FROM events e
        JOIN event_types t ON e.event_type = t.event_type
        WHERE e.ts >= ?1
        ",
        [fifteen_ago],
        |row| row.get(0),
    ).unwrap();

    // YTD
    let start_of_year = Utc::now()
        .with_month(1).unwrap()
        .with_day(1).unwrap()
        .with_hour(0).unwrap()
        .with_minute(0).unwrap()
        .with_second(0).unwrap()
        .timestamp();

    let ytd: f64 = conn.query_row(
        "
        SELECT COALESCE(SUM(e.units * t.dollars_per_unit), 0)
        FROM events e
        JOIN event_types t ON e.event_type = t.event_type
        WHERE e.ts >= ?1
        ",
        [start_of_year],
        |row| row.get(0),
    ).unwrap();

    Totals { last_15m, ytd }
}

This is the real math behind your bouncing dollar.

#[tauri::command]
fn poll_git(state: tauri::State<AppState>, repo_path: String) {
    use std::process::Command;

    let output = Command::new("git")
        .arg("-C")
        .arg(repo_path)
        .arg("rev-list")
        .arg("--count")
        .arg("HEAD")
        .output()
        .unwrap();

    let count_str = String::from_utf8_lossy(&output.stdout);
    let commit_count: i32 = count_str.trim().parse().unwrap();

}

⭐ PART 5 — VS Code event capture (YES, via OS APIs)

Simplest first event: VS Code is focused.
Windows example (Rust):
rust

use windows::Win32::UI::WindowsAndMessaging::{GetForegroundWindow, GetWindowTextW};

#[tauri::command]
fn poll_active_window(state: tauri::State<AppState>) {
    let hwnd = unsafe { GetForegroundWindow() };

    let mut buf = [0u16; 512];
    unsafe { GetWindowTextW(hwnd, &mut buf); }
    let title = String::from_utf16_lossy(&buf);

    if title.contains("Visual Studio Code") {
        log_event(state, "deep_work_block".into(), 1.0);
    }
}

You run this every 30 seconds.
If VS Code is focused → log deep work.

This is the first real behavioral signal.
